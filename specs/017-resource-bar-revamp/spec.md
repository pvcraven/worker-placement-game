# Feature Specification: Resource Bar Revamp

**Feature Branch**: `017-resource-bar-revamp`  
**Created**: 2026-04-22  
**Status**: Draft  
**Input**: User description: "For this round, we are going to update the bottom panel. First, in the area that makes the cards and markers, let's also create images of the resources (squares, guitarists, etc.) as pngs. Then in that bottom panel change the background color to be the same parchment color as the cards. Then instead of drawing squares, draw those sprites. (use a sprite list, and don't recreate the spritelist each frame). Draw the text in the same color/font as the cards. Use the intrigue/quest icons for those status points. Use the marker for the player next to 'workers left'."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Generate Resource Icon PNGs (Priority: P1)

The card image generator produces PNG images for each resource type (guitarists, bass players, drummers, singers, coins) so they can be used as sprites in the bottom panel. These icons use the same colored-square and gold-circle style established on quest/space cards.

**Why this priority**: The resource icon PNGs must exist before the bottom panel can display them as sprites. This is a prerequisite for all other stories.

**Independent Test**: Run the card generator and verify PNG files are created in the expected output directory with correct colors and shapes.

**Acceptance Scenarios**:

1. **Given** the card generator is run, **When** resource icon generation completes, **Then** PNG files exist for guitarists (orange square), bass players (black square), drummers (purple square), singers (white square), and coins (gold circle), each with a black outline.
2. **Given** the generated PNGs, **When** viewed at native resolution, **Then** they are crisp and appropriately sized for use as small inline icons in the resource bar.

---

### User Story 2 - Restyle Bottom Panel Background and Text (Priority: P2)

The bottom resource panel background changes from the current dark color (25, 25, 35) to the same parchment/tan color used on card backgrounds. All text in the panel uses the same font and color as card text (dark brown on parchment, matching the card generator style).

**Why this priority**: The visual restyling sets the foundation before swapping in sprite-based icons, and can be tested independently by simply viewing the panel.

**Independent Test**: Launch the game client and verify the bottom panel has a parchment-colored background and dark card-style text instead of white-on-dark.

**Acceptance Scenarios**:

1. **Given** the game is running, **When** the player views the bottom resource bar, **Then** the background is the same parchment color used on card images.
2. **Given** the restyled panel, **When** reading resource labels and counts, **Then** text is rendered in the same dark color and font style used on generated card images.

---

### User Story 3 - Display Resource Sprites Instead of Drawn Squares (Priority: P3)

The bottom panel replaces the currently drawn colored rectangles (swatches) with sprite images loaded from the generated PNGs. A SpriteList is used and is only rebuilt when the data changes, not every frame.

**Why this priority**: This is the core visual upgrade — swapping drawn primitives for polished sprite icons — but depends on the PNGs existing (US1) and the background being updated (US2).

**Independent Test**: Launch the game and verify that resource icons in the bottom bar are rendered as sprite images, and that scrolling/resizing does not cause visible flickering or performance issues.

**Acceptance Scenarios**:

1. **Given** resource icon PNGs exist, **When** the bottom panel renders, **Then** each resource type displays its corresponding sprite image instead of a drawn rectangle.
2. **Given** the SpriteList is built, **When** the panel redraws on subsequent frames with no data change, **Then** the SpriteList is reused (not recreated).
3. **Given** the player's resource counts change, **When** the panel updates, **Then** the SpriteList rebuilds to reflect the new state.

---

### User Story 4 - Use Card Icons for Quest and Intrigue Counts (Priority: P4)

The top row of the bottom panel uses the existing quest card icon (Q) and intrigue card icon (I) — already generated by the card icon system — next to the quest and intrigue count numbers, replacing plain text labels.

**Why this priority**: Enhances visual consistency with the card icon system already used on space cards, but is an incremental polish step.

**Independent Test**: Launch the game and verify the quest open/done and intrigue count entries in the bottom bar show the miniature card icons (Q and I) beside their numbers.

**Acceptance Scenarios**:

1. **Given** the bottom panel is displayed, **When** viewing quest counts, **Then** a quest card icon (Q, dark red back) appears next to "Open" and "Done" counts.
2. **Given** the bottom panel is displayed, **When** viewing intrigue count, **Then** an intrigue card icon (I, dark gray back) appears next to the count.

---

### User Story 5 - Show Player Marker Next to Workers Left (Priority: P5)

The "Workers left" entry in the bottom panel displays the player's colored worker marker sprite beside the worker count, so the player can immediately identify their color.

**Why this priority**: Small polish enhancement that uses existing marker PNGs.

**Independent Test**: Launch the game and verify a colored marker image matching the player's color appears next to the "Workers left" label.

**Acceptance Scenarios**:

1. **Given** the player has a color assignment, **When** the bottom panel renders, **Then** the player's worker marker sprite is displayed next to the workers left count.

---

### Edge Cases

- What happens when the window is resized very small? Resource sprites and icons should scale down gracefully without overlapping.
- What happens if a resource icon PNG is missing? The panel should fall back to drawing a colored rectangle (current behavior) rather than crashing.
- What happens when a player has zero of a resource? The sprite and count should still display (showing 0).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: The card generator MUST produce PNG icon images for each resource type: guitarists (orange square), bass players (black square), drummers (purple square), singers (white square), and coins (gold circle), all with black outlines.
- **FR-002**: The bottom resource panel MUST use a parchment-colored background matching the card image background color.
- **FR-003**: All text in the bottom resource panel MUST use the same font and color style as text on generated card images (dark color on parchment).
- **FR-004**: The bottom panel MUST display resource counts using sprite images loaded from the generated PNGs instead of drawn rectangles.
- **FR-005**: The bottom panel MUST use a SpriteList for resource icons that is only rebuilt when underlying data changes, not on every frame.
- **FR-006**: The bottom panel MUST display quest card icons (Q) next to quest-related counts and intrigue card icons (I) next to intrigue count.
- **FR-007**: The bottom panel MUST display the player's colored worker marker sprite next to the "workers left" count.
- **FR-008**: If a resource icon PNG is missing, the panel MUST fall back to drawing a colored shape instead of crashing.

### Key Entities

- **Resource Icon**: A small PNG image representing a single unit of a resource type (guitarist, bass player, drummer, singer, coin).
- **Card Icon**: A miniature playing-card-shaped image with a letter (Q or I) used to represent quest or intrigue card counts.
- **Worker Marker**: An existing player-colored marker PNG used to identify the player's color.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: All five resource type icon PNGs are generated successfully by the card generator.
- **SC-002**: The bottom panel visually matches the parchment card aesthetic — background color, text font, and text color are consistent with card images.
- **SC-003**: Resource icons render as sprites in the bottom panel with no per-frame sprite list recreation when data is unchanged.
- **SC-004**: Quest and intrigue counts display their respective card icons alongside numeric values.
- **SC-005**: The player's worker marker appears next to the workers-left count and matches the player's assigned color.
- **SC-006**: The bottom panel renders correctly across different window sizes without overlapping or clipping.

## Assumptions

- The existing card icon system (`_draw_quest_card_icon`, `_draw_intrigue_card_icon` in generate_cards.py) will be reused to generate standalone icon PNGs.
- The parchment color is the same value used in the card generator for card backgrounds.
- Worker marker PNGs already exist at `client/assets/card_images/markers/worker_{color}.png` and do not need to be regenerated.
- The resource icon PNGs will be stored in the existing `client/assets/card_images/` directory structure.
- The resource bar will continue to occupy the same screen region (bottom of window, same height ratio).
